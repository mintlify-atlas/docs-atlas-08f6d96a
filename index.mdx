AI Balancer is a FastAPI-based API gateway that routes your LLM requests across multiple providers — Grok, Cerebras, Gemini, and OpenRouter — through a single unified interface. It handles provider selection automatically using Round Robin load balancing, protects your application from cascading failures with an adaptive Circuit Breaker, and secures access with per-client API key management.

## What AI Balancer does

Instead of integrating with multiple LLM providers individually, you send all your requests to AI Balancer. It selects the best available provider, handles retries, and streams responses back to your client — all through a consistent API.

  <Card title="Round Robin load balancing" icon="arrows-spin" href="/concepts/load-balancing">

## Get started in 3 steps

<Steps>
  <Step title="Install and configure">
    Clone the repository, install dependencies, and copy `.env.example` to `.env` with your provider API keys.

    ```bash
    git clone https://github.com/Pyhton-Lab/api-gateway-ai.git
    cd api-gateway-ai
    pip install -r requirements.txt
    cp .env.example .env
    ```
  </Step>
  <Step title="Start the server">
    Run the gateway. On first start, an admin API key is printed to the console — save it.

    ```bash
    python main.py
    # ⚠️ ADMIN API KEY: sk-abc123... ⚠️ (SAVE IT - only shown once)
    ```
  </Step>
  <Step title="Send your first request">
    Use your admin key to create a client key, then call the chat endpoint.

    ```bash
    curl -X POST http://localhost:8000/v1/chat \
      -H "X-API-Key: YOUR_API_KEY" \
      -H "Content-Type: application/json" \
      -d '{"messages": [{"role": "user", "content": "Hello!"}]}'
    ```
  </Step>
</Steps>
